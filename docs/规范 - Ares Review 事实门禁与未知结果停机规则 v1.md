# 规范 - Ares Review 事实门禁与未知结果停机规则 v1

## 核心原则 (Core Principle)
**Truth > Completeness（真相高于完整性）**
不知道就说不知道。绝不为了“完成任务”而捏造事实。

## 触发停机 (Halt) 的条件
在执行 `Review` (赛后复盘) 阶段，若遇到以下任一情况，Ares **必须立即终止** 复盘生成：
1. **接口返回 Unknown**：爬虫或 API 获取的比分为空、未开奖或显示 Unknown。
2. **官方页未更新**：新浪彩票、500彩票网等基准数据源尚未发布正式赛果。
3. **比分缺失**：部分场次（如因天气推延、腰斩）没有最终确定的全场比分。
4. **多源冲突**：两个以上数据源对同一场次的比分或赛果存在不一致。
5. **无法说清结果来源**：如果 Ares 无法明确指出赛果是从哪个真实的检索源（URL/新闻）中提取的。

## 停机后的合规输出 (Allowed Output)
触发停机后，Ares **仅允许** 在控制频道输出以下类似的话术，并结束对话：
- “等待赛果更新”
- “当前关键赛果未确认，暂停正式复盘”
- “待官方结果更新后重新执行”

## 严禁行为 (Strict Prohibitions)
- **严禁** 将 Prematch 的“预测”倒写成 Post-match 的“事实”。
- **严禁** 在未核实官方比分时，输出“命中区”、“避雷成功区”或“全红”等词汇。
- **严禁** 编造比分、绝杀时间或比赛过程。
- **严禁** 生成不包含真实赛果的、纯逻辑推演的虚假 Review 报告。
